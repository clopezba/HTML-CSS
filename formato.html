Formato de HTML

El formato básico que presenta un documento HTML es el que se muestra en la figura.

  • En HTML5 es obligatorio indicar la declaración del tipo de documento en la primera línea.
  • La etiqueta html indica el inicio y final del documento.
  • El primer elemento de HTML es head, es opcional pero recomendado ponerlo. Incluye los metadatos, información sobre el documento.
  • Body es el cuerpo del documento. Todo el contenido que se mostrará se incluye aquí.

El formato de etiqueta en HTML es el mismo que en XML: ‹elemento›.
De la misma forma que la etiqueta de cierre es ‹/elemento›.
Pero sí hay una diferencia con respecto a XML ya que los elementos vacíos no llevan etiqueta de cierre.

En el caso de los atributos, también se indican igual que en lenguaje XML, pero en este caso no es necesario poner las comillas (" "). Sí es interesante ponerlas cuando haya espacios en blanco o caracteres raros.

El atributo "lang" no es obligatorio, pero sí recomendado. Indica el idioma del texto del documento. Este atributo no necesariamente se debe poner en la etiqueta ‹html›, sino que se puede poner en cualquier otro elemento (ej. poner una cita en otro idioma).

Título

El elemento ‹title› representa el título principal de todo el documento HTML. Éste se mostrará en el nombre de la ventana/pestaña del navegador o cuando se guarde como favorito o en marcadores.
Es un metadato, por lo que no se mostrará en el contenido de la página, y se debe incluir dentro de ‹head›.


Metadatos

Los metadatos son los datos sobre el documento que no se muestran. Se indican con el elemento ‹meta› y existen muchos tipos distintos.
Los principales usos del elemento ‹meta› son charset y name.

Charset
Especifica la codificación de caracteres del documento. La más habitual es utf-8, que es un conjunto de caracteres universal que incluye casi todos los caracteres de casi cualquier idioma humano.
Name
Sirve para definir otras datos del documento, como el autor o una descipción de la página. Para ello daremos un nombre ("name") y un valor ("content").


Body

Es la parte visible del documento, donde se incluye todo el contenido que se quiere mostrar. Dentro del cuerpo podemos encontrar diferentes elementos:

Títulos
Llamados en inglés heading, son los títulos del interior del documento. Pueden incluirse en las diferentes secciones, artículos o encabezados.
Hay hasta seis niveles de títulos que se indican con las etiquetas ‹h1› hasta ‹h6›.

Artículo
Es un contenedor de bloques de contenido que se consideran independientes del sitio web pueden ser vistos, reutilizados y distribuidos pòr separado. Es habitual encontrar este elemento encerrando artículos, entradas de blogs o mensajes en un foro.
Este elemento se indica con la etiqueta ‹article›.

Sección
El elemento ‹section› es un elemento de seccionamiento genérico, diseñado para contener una parte de un documento temáticamente definida. Suele utilizarse para dividir un documento extenso en capítulos, separar las diferentes pestañas en una aplicación o dividir temáticamente las secciones de un documento.

Navegación
Se trata de los menús de navegación dentro de un documento HTML. Se indica con el elemento ‹nav›, y puede incluirse en el encabezado (header), el pie de página (footer) o fuera de estas secciones.

Inciso
Se utiliza la etiqueta ‹aside› y sirve para dar información complementaria. El contenido del elemento debe estar relacionado con el contenido circundante, pero no ser completamente necesario para su comprensión.
Es habitual que se sitúe en un lateral del documento HTML.

Encabezado
Representa un grupo de ayudas introductorias o de navegación. Puede contener algunos elementos de encabezado, así como también un logo, un formulario de búsqueda o un nombre de autor, entre otros.
Este elemento se indica con la etiqueta ‹header›.

Pie de página
El elemento ‹footer› representa un pie de página para el contenido de sección más cercano o el elemento raíz de sección. Normalmente contiene información acerca del autor de la seción, datos de derechos de autor o enlaces a documentos relacionados.

Dirección
La dirección, que utiliza la etiqueta ‹address›, representa la información de contacto del cuerpo o artículo más cercano. Si se encuentra en el cuerpo, la información de contacto se aplica a todo el documento.
Habitualmente un elemento ‹address› peude ser colocado dentro del elemento ‹footer› de la sección actual, si lo hay.

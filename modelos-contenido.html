Modelos de contenido

Cada elemento HTML es miembro de una o más categorías de contenido. Estas categorías agrupan elementos que comparten características comunes.

Categorías de contenido principal

Contenido de metadatos (metadata)
Los elementos pertenecientes a la categoría contenido de metadatos modifican la presentación o el comportamiento del resto del documento o establecen enlaces a otros documentos.
Contenido dinámico (flow)
Los elementos que pertenecen a la categoria de contenido dinámico suelen contener texto o contenido encajado. Abarca básicamento todos los elementos.
Contenido seccionado (sectioning)
Los elementos que pertenecen al modelo de contenido de secciones crean una sección en el esquema actual.
Contenido del encabezado (heading)
El contenido del encabezado define el título de una sección, ya sea que esté marcado por un elemento de contenido seccionado explícito o definido implícitamente por el contenido del encabezado en sí mismo.
Contenido textual (phrasing)
El contenido textual define el texto y el marcado que contiene. Las series de contenido textual forman párrafos.
Contenido encajado (embedded)
El contenido encajado importa otro recurso o inserta contenido al documento, por ejemplo, contenido multimedia.
Contenido interactico (interactive)
El contenido interactivo incluye elementos diseñados específicamente para la interacción del usuario.

Categorias secundarias

Contenido tangible (palpable: flow o phrasing)
El contenido es palpable cuando no está vacío ni oculto; es contenido que se presenta y es sustantivo. Los elementos cuyo modelo es dinámico (flow) o contenido textual (phrasing) deben tener, por lo menos, un nodo que sea palpable.
Soporte de scripts (script-supporting)
Los elementos de soporte de scripts son elementos que no contribuyen directamente a la salida renderizada de un documento. En cambio, sirven para admitir scripts, ya sea conteniendo o especificando directamente el código del script, o especificando datos que serán utilizados por los scripts.
Las categorías que admiten scripts son: script y template.
Modelo de contenido transparente (transparent)
Si un elemento tiene un modelo de contenido transparente, entonces su contenido debe estar estructurado de manera que sea HTML 5 válido, incluso si el elemento transparente fuera eliminado y reemplazado por elementos secundarios.
Párrafo (paragraph)
Un párrafo es normalmente una serie de contenido textual que forma un bloque de texto con una o más oraciones que tratan un tema en particular, como en la tipografía, pero también se puede usar para agrupaciones temáticas más generales.
